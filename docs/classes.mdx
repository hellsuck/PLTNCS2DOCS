# Classes

Plutonium uses an object-oriented approach for certain data structures and game objects.

---

## `ConVar`

The `ConVar` class represents a game console variable. Objects of this class are returned by [`get_cvar()`](/globals#get_cvar).

<Accordion title="Methods" defaultOpen>
  <Property name="get_float" type="function" required>
    Returns the float value of the CVar.
  </Property>

  ---

  <Property name="set_float" type="function" required>
    <Property name="val" type="float" required>
      The new float value.
    </Property>
    Sets the float value of the CVar.
  </Property>

  ---

  <Property name="get_int" type="function" required>
    Returns the integer value of the CVar.
  </Property>

  ---

  <Property name="set_int" type="function" required>
    <Property name="val" type="int" required>
      The new integer value.
    </Property>
    Sets the integer value of the CVar.
  </Property>
</Accordion>

> [!WARNING]
> Be careful when writing to certain ConVars as they can negatively affect your **trust factor** and overall game experience.

```lua
local sensitivity = get_cvar("sensitivity")
print("Original: " .. sensitivity:get_float())
sensitivity:set_float(2.0)
```

## `Vector3`

The `Vector3` class represents a 3D coordinate or vector.

<Accordion title="Properties">
  <Property name="x" type="float" required>
    X coordinate.
  </Property>

  ---

  <Property name="y" type="float" required>
    Y coordinate.
  </Property>

  ---

  <Property name="z" type="float" required>
    Z coordinate.
  </Property>
</Accordion>

<Accordion title="Methods">
  <Property name="new" type="function" required>
    Creates a new `Vector3` instance.
  </Property>

  ---

  <Property name="distance" type="function" required>
    <Property name="other" type="Vector3" required>
      The other Vector3 to calculate distance to.
    </Property>
    Calculates the distance to another `Vector3` object.
  </Property>
</Accordion>

```lua
local pos1 = Vector3.new(100, 100, 0)
local pos2 = Vector3.new(200, 200, 0)
local dist = pos1:distance(pos2)
print("Distance: " .. dist)
```

## `CUserCmd`

The `CUserCmd` class represents a user command sent to the server.

<Accordion title="Methods" defaultOpen>
  <Property name="set_sub_tick_angle" type="function" required>
    <Property name="angView" type="Vector3" required>
      The desired view angle.
    </Property>
    Sets the subtick view angle.
  </Property>
</Accordion>

```lua
register_callback("create_move", function(cmd)
    local my_angle = Vector3.new(89, 180, 0)
    cmd:set_sub_tick_angle(my_angle)
end)
```

> [!WARNING]
> Writing **illogical or impossible** values and very fast changes can trigger **neural anticheat detection**. Always ensure your logic is consistent with human-like input if used for aim-assist features.
